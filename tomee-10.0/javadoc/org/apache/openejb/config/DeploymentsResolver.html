      • loadFromClasspath

        public static java.util.List<java.net.URL> loadFromClasspath(java.lang.ClassLoader classLoader)
        The algorithm of OpenEJB deployments class-path inclusion and exclusion is implemented as follows: 1- If the string value of the resource URL matches the include class-path pattern Then load this resource 2- If the string value of the resource URL matches the exclude class-path pattern Then ignore this resource 3- If the include and exclude class-path patterns are not defined Then load this resource The previous steps are based on the following points: 1- Include class-path pattern has the highest priority This helps in case both patterns are defined using the same values. This appears in step 1 and 2 of the above algorithm. 2- Loading the resource is the default behaviour in case of not defining a value for any class-path pattern This appears in step 3 of the above algorithm.
