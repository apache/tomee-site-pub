org.apache.openejb.util

Class Pool<T>

  • java.lang.Object
    • org.apache.openejb.util.Pool<T>

  • public class Pool<T>
extends java.lang.Object
    Any successful pop() call requires a corresponding push() or discard() call. A pop() call that returns null is considered successful. A null indicates that the calling code has a permit to create a poolable object and call push(Object). This is the only situation in which that method may be called. To simply fill the pool without a corresponding pop(), the add() method must be used. This method will attempt to aquire a permit to add to the pool.
